# Capability Debt Report

Purpose: define the operator next-action report for capability governance debt.
Governance scope: [OCE, RAG, CDCV, CQTE, UWMA, SRCA, PRS]
Dependencies: capability passports, gate template registry, evidence passports,
sandbox-to-live promotion paths, debt report schema, and debt report validator.
Invariants: every capability has one debt row; missing approval, evidence,
replay, rollback, promotion, and live-action blockers are explicit; debt rows
never grant execution authority.

## Architecture

The debt report turns the previous reusable systems into a clear action list:

1. Capability Passport supplies identity, family, unlock level, and status.
2. Gate Template Registry supplies reusable governance gate semantics.
3. Evidence Passport supplies missing proof, approval, replay, and rollback state.
4. Sandbox-to-Live Promotion Path supplies blocked stage controls.
5. Capability Debt Report emits one row per capability plus top debt items.

The read model is generated by:

```text
mcoi/mcoi_runtime/app/capability_debt_report.py
```

It emits:

```text
examples/capability_debt_report.foundation.json
```

## Debt Categories

| Category | Meaning |
| --- | --- |
| `evidence` | Required evidence or receipt is missing. |
| `approval` | Governed operator or policy approval is missing. |
| `rollback` | Rollback, compensation, or recovery proof is missing. |
| `replay` | Deterministic replay evidence is missing. |
| `promotion` | Sandbox-to-live stage controls are blocked. |
| `live_action` | Live action remains disabled. |

## Severity

| Severity | Meaning |
| --- | --- |
| `critical` | Capability is blocked or certification is not active. |
| `high` | Required evidence, approval, or rollback proof blocks promotion. |
| `medium` | Replay or promotion controls need closure. |
| `low` | Live action remains disabled by Foundation Mode or policy. |

## Governance Boundary

These fields must remain fixed:

```text
debt_report_is_not_execution_authority = true
live_execution_enabled = false
debt_row_is_not_execution_authority = true
live_action_enabled = false
```

The debt report chooses next evidence work. It does not grant tool authority,
connector access, external send permission, live action, production readiness,
or terminal closure.

Approval debt is projected from the evidence passport approval packet. When a
capability requires `gate.approval.required`, the debt item must name the
missing approval gate, `approval_decision_receipt`, required approval inputs,
and the blocked approval action. This keeps approval work actionable while
remaining a read-only planning record.

Rollback debt is projected from the evidence passport rollback packet. When
rollback or recovery evidence is missing, the debt item must name concrete refs
such as `recovery_evidence_missing`, `rollback_capability`,
`compensation_capability`, `failure_review_receipt`, and
`rollback_or_recovery_evidence`. This keeps recovery work actionable while
remaining a read-only planning record.
