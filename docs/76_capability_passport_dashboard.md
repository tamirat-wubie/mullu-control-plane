# Capability Passport Dashboard

Purpose: define the foundation-stage operator read model for capability
passport status.
Governance scope: [OCE, RAG, CDCV, CQTE, UWMA, SRCA, PRS]
Dependencies: capability passports, gate template registry, dashboard schema,
and dashboard validator.
Invariants: the dashboard is read-only, hides internal gate and schema detail
from operator cards, and never grants execution authority.

## Architecture

The Capability Passport Dashboard turns passport state into simple operator
visibility:

| Operator label | Meaning |
| --- | --- |
| Ready | The capability has enough governed evidence for its current allowed action surface. |
| Prepare-only | The capability can prepare work but must not execute live effects. |
| Needs approval | The capability requires operator or policy approval before execution. |
| Blocked | The capability is suspended, retired, or missing a hard governance requirement. |
| Evidence missing | The capability needs evidence intake or verification before promotion. |
| Live action disabled | The capability can be inspected or prepared, but live action remains disabled. |

The read model is generated by
`mcoi/mcoi_runtime/app/capability_passport_dashboard.py`.

It consumes:

1. `examples/capability_passports.foundation.json`
2. `examples/gate_template_registry.foundation.json`

It emits:

1. `examples/capability_passport_dashboard.foundation.json`
2. operator status tiles
3. operator status lanes
4. family rows
5. governance health counters

## Operator Surface

Operator cards expose only:

1. capability identity
2. capability family
3. current operator status
4. current unlock level
5. allowed action names
6. blocked action count
7. required receipt count
8. rollback status
9. next unlock step

Operator cards do not expose raw gate IDs, schema refs, admission internals, or
source-file paths.

## Governance Boundary

The dashboard is not execution authority.

The following fields must remain fixed:

```text
dashboard_is_not_execution_authority = true
live_execution_enabled = false
governance_health.operator_view_hides_internal_gate_ids = true
governance_health.operator_view_hides_schema_refs = true
```

Execution authority still belongs to governed capability packs, UAO admission,
gate templates, approval policy, evidence verification, receipts, and rollback
policy.
