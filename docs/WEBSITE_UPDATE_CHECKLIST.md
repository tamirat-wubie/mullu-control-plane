# Website Update Checklist

Purpose: define the public-site update requirements for introducing `Mullu` as the flagship product by Mullusi.
Governance scope: homepage, product page, docs landing page, footer, pricing, app entry, and launch constraints.
Dependencies: `docs/PUBLIC_LAUNCH_COPY.md`, `docs/PRODUCT_IDENTITY.md`, `docs/PUBLIC_NAMING_READINESS.md`, `docs/DOMAIN_ACQUISITION_PLAN.md`.
Invariants: `Mullu` is the product; `Mullusi` is the company and governance authority; public paid launch is blocked until clearance gates close.

## Homepage Requirements

| Section | Requirement |
| --- | --- |
| Hero eyebrow | `Mullu, by Mullusi` |
| Hero headline | `Mullu` or the one-line public anchor |
| Tagline | `Symbols are atomic. Meaning is relational. Traversal is governed. Judgment is earned.` |
| Supporting copy | Governed symbolic intelligence for personal, enterprise, developer, and deployment work |
| Primary action | Waitlist, private beta, or request access until clearance closes |
| Secondary action | Read the product architecture or research |

## Product Page Requirements

1. Use `Mullu` after the first-reference line.
2. Explain surfaces: Mullu, Mullu Inspect, Mullu CLI, Mullu Code, Mullu Desk, Mullu Control Plane.
3. Keep `Mullusi` in footer, governance, audit, research, and company contexts.
4. Avoid generic product names: `Mullusi Handler`, `Mullusi Work`, `Mullusi Operator`, `Mullu AI`.
5. Avoid overpromising live production status before deployment witness exists.

## Trust and Governance Copy

Use:

```text
Every governed action is auditable, budget-controlled, policy-enforced, and
approval-gated when risk requires it.
```

Use:

```text
Mullu preserves the causal chain behind work: who asked, what changed, which
constraints applied, what evidence closed the task, and what memory was admitted
afterward.
```

```text

```


## Pre-Publish Checks

1. Clearance gates closed or page marked private beta/waitlist.
2. Product domain or fallback route verified.
3. Public copy uses `Mullu`.
4. Company copy uses `Mullusi`.
5. Technical docs use `Mullu Platform` only where appropriate.
6. Admin surface uses `Mullu Control Plane`.
7. No stale public names appear.
8. No external production runtime is claimed without deployment witness.
