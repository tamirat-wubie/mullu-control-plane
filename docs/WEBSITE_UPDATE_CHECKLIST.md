# Website Update Checklist

Purpose: define the public-site update requirements for introducing `Mullu Govern` as the public product by Mullusi.
Governance scope: homepage, product page, docs landing page, footer, pricing, app entry, and launch constraints.
Dependencies: `docs/PUBLIC_LAUNCH_COPY.md`, `docs/PRODUCT_IDENTITY.md`, `docs/PUBLIC_NAMING_READINESS.md`, `docs/DOMAIN_ACQUISITION_PLAN.md`.
Invariants: `Mullu Govern` is the public product; `Mullu` is the suite/family; `Mullusi` is the company and governance authority; public paid launch is blocked until clearance gates close.

## Homepage Requirements

| Section | Requirement |
| --- | --- |
| Hero eyebrow | `Mullu Govern, by Mullusi` |
| Hero headline | `Mullu Govern` or the one-line public anchor |
| Tagline | `Symbols are atomic. Meaning is relational. Traversal is governed. Judgment is earned.` |
| Supporting copy | Governed symbolic execution for workflows, approvals, budgets, audit trails, lineage, deployment controls, and proof-backed actions |
| Primary action | Foundation-stage proof-boundary action only until clearance and witness gates close |
| Secondary action | Read the product architecture or research |

## Product Page Requirements

1. Use `Mullu Govern` after the first-reference line.
2. Explain surfaces: Mullu Govern, Mullu Proof, Mullu Ledger, Mullu Inspect, Mullu CLI, Mullu Code, Mullu Desk, and Mullu Control Plane.
3. Keep `Mullusi` in footer, governance, audit, research, and company contexts.
4. Avoid generic product names: `Mullusi Handler`, `Mullusi Work`, `Mullusi Operator`, `Mullu Generic`.
5. Avoid overpromising live production status before deployment witness exists.

## Pre-Publish Checks

1. Clearance gates closed or page marked foundation-stage with no access, waitlist, or beta invitation.
2. Product domain or fallback route verified.
3. Public copy uses `Mullu Govern`.
4. Company copy uses `Mullusi`.
5. Technical docs use `Mullu Platform` only where appropriate.
6. Admin/internal technical surface uses `Mullu Control Plane`.
7. No stale public names appear.
8. No external production runtime is claimed without deployment witness.
