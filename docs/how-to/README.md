# How-To Guides

> **In one box:** Short, task-focused recipes: "I want to do *X* — give me the
> steps." Unlike a [tutorial](../tutorials/01_first_governed_task.md) (which
> teaches a beginner slowly), a how-to assumes you already know the basics and
> just need the steps for one specific goal. No prior knowledge of *this page*
> is assumed, but each guide may assume you've done the tutorial once.

If a word confuses you, it's one click away in the [Glossary](../GLOSSARY.md).

---

## Adding a new how-to

Copy [`../_TEMPLATE.md`](../_TEMPLATE.md), set its TYPE to **How-to**, name the
file `verb-the-goal.md` (e.g. `set-a-spend-budget.md`), and add a row to the
table above. Keep it to the steps — motivation belongs in
[explain/](../explain/), full rules belong in [reference/](../reference/).
