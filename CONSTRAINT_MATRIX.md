# Domain Adapter Constraint Matrix

Generated by `mcoi/tools/audit_constraint_matrix.py` from the live adapter registry (15 adapters as of generation time). Each adapter is probed with its minimal request plus a battery of single-flag mutations and bool-pair stress combinations; the table below aggregates every `(constraint_domain, violation_response)` tuple emitted across all probes.

**Why this exists.** The universality claim of the domain adapter framework is structural: every adapter projects domain actions into the same `UniversalRequest` shape. The table makes the claim machine-checkable — adding a new adapter regenerates this file; the test [`test_constraint_matrix.py`](mcoi/tests/test_constraint_matrix.py) fails if the checked-in file drifts from what the tool produces.

## Cross-adapter patterns

### Universal pattern — `<vertical>_correctness → block`

Every one of the 15 adapters emits at least one constraint of the form `<vertical>_correctness` with `violation_response = block`. This is the acceptance-criteria contract: each `acceptance_criteria` entry on a domain request becomes a `block` constraint in the universal envelope, which the L9 layer of UCJA enforces. The exact names vary by domain vocabulary:
- `business_process` → `business_correctness`
- `construction` → `construction_correctness`
- `cybersecurity` → `secops_correctness`
- `education` → `instructional_correctness`
- `energy` → `energy_correctness`
- `environmental` → `environmental_correctness`
- `finance` → `financial_correctness`
- `healthcare` → `clinical_correctness`
- `insurance` → `insurance_correctness`
- `legal` → `legal_correctness`
- `logistics` → `logistics_correctness`
- `manufacturing` → `production_correctness`
- `public_sector` → `civic_correctness`
- `scientific_research` → `research_correctness`
- `software_dev` → `software_correctness`

### Universal by literal name — emitted by every adapter

_None — every adapter uses its own vocabulary; the `<vertical>_correctness` pattern above is the structural analog._

### Graded — same constraint domain emits different violation responses depending on context

These are the kind-inverting and emergency-relaxing constraints (see each adapter's source for the precise rule):

- `deadline (legal → block/escalate)`
- `exceedance (environmental → block/escalate)`
- `public_comment (public_sector → block/escalate)`
- `safety_freeze (construction → block/escalate)`
