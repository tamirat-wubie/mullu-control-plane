# Mullusi Agentic Service Harness Read-Only Status Route Design

## Objective

Define the first design-only route contract for displaying Agentic Service Harness status from already validated read-model and persistence rehearsal artifacts. This artifact does not implement a route.

Solver outcome: `SolvedVerified` for design boundary after validator passage; implementation evidence is tracked by the separate read-only status route validator.

## Route Contract

| Field | Value |
| --- | --- |
| Route id | `agentic_service_harness_status_read_model` |
| Method | `GET` |
| Path | `/api/v1/harness/status` |
| Companion method | `HEAD` only if framework-generated by the read route |
| Route class | `read_model` |
| Authority class | `read_only` |
| Source projection | `examples/agentic_service_harness_read_models.foundation.json` |
| Persistence witness | `scripts/validate_agentic_service_harness_read_model_persistence.py` |
| Status before implementation | `AwaitingEvidence` |

No route decorator, router registration, FastAPI handler, persistence adapter, dashboard screen, external adapter call, branch write, pull-request creation, merge, deployment, DNS mutation, secret mutation, or destructive operation is admitted by this design.

## State Behavior

| Condition | Response status | Required blocker |
| --- | --- | --- |
| Source read model valid and persistence rehearsal valid | `AwaitingEvidence` | `read_only_status_route_implementation_pending` |
| Source read model missing | `AwaitingEvidence` | `missing_read_model_source` |
| Persistence rehearsal missing or failing | `AwaitingEvidence` | `missing_persistence_rehearsal` |
| Tenant or project scope missing | `AwaitingEvidence` | `missing_tenant_or_project_scope` |
| Any high-risk authority flag true | `GovernanceBlocked` | `high_risk_authority_not_allowed` |
| Any secret-like value serialized | `GovernanceBlocked` | `secret_value_serialization_not_allowed` |
| Any terminal closure claim present | `GovernanceBlocked` | `terminal_closure_claim_not_allowed` |

## Error And Staleness Behavior

1. Missing read-model source returns a governed status projection with `AwaitingEvidence`.
2. Invalid schema, invalid replay, stale evidence, or missing tenant/project scope returns a governed status projection with explicit blockers.
3. Secret-like values, mutation authority, terminal closure claims, or high-risk authority return `GovernanceBlocked`.
4. The design does not define live endpoint status codes. Status-code behavior waits for the later route implementation artifact.
5. No raw command logs, raw diffs, raw credentials, tokens, private keys, webhook secrets, or unredacted stdout/stderr may appear inline.
